# 🛠️ Acer Aspire 5742G BIOS Unlock (Advanced & Power Menus) + Custom Logo

В данном репозитории находятся дампы BIOS для ноутбука Acer Aspire 5742G, а также подробная пошаговая инструкция по самостоятельной разблокировке скрытых инженерных меню (Advanced и Power) и замене загрузочного логотипа.

⚠️ **КРИТИЧЕСКИЕ ПРЕДУПРЕЖДЕНИЯ (ЧИТАТЬ ОБЯЗАТЕЛЬНО!)** ⚠️
1. **ТОЛЬКО ПРОГРАММАТОР:** Прошивка модифицированного BIOS должна производиться **СТРОГО** через аппаратный программатор (например, CH341A) с прищепкой или через выпаивание чипа памяти. 
2. **НЕТ ПРОШИВКЕ ИЗ WINDOWS:** Попытка прошить этот мод через штатные утилиты (InsydeFlash и др.) из-под ОС приведет к ошибке контрольных сумм / проверке цифровой подписи и гарантированному **«окирпичиванию»** ноутбука.
3. **ОБЕСТОЧЬТЕ ПЛАТУ:** Перед подключением прищепки программатора к чипу **ОБЯЗАТЕЛЬНО** отключите блок питания и снимите аккумуляторную батарею с ноутбука. 
4. **СДЕЛАЙТЕ БЭКАП:** Прежде чем что-то зашивать, считайте свой оригинальный BIOS через программатор минимум 2-3 раза. Сохраните эти файлы и проверьте, чтобы они были абсолютно идентичны (совпадали хэш-суммы). Если файлы разные — прищепка установлена криво, считывать и шить так нельзя!
5. **ЗАПАСНОЙ ПК:** Не приступайте к прошивке модифицированного BIOS, если у вас нет под рукой второго рабочего компьютера/ноутбука. В случае ошибки вам понадобится второе устройство, чтобы прошить оригинальный дамп обратно.

> **ВСЕ ДЕЙСТВИЯ ВЫ ВЫПОЛНЯЕТЕ ИСКЛЮЧИТЕЛЬНО НА СВОЙ СТРАХ И РИСК!** Автор репозитория не несет ответственности за вышедшее из строя оборудование.

## 📁 Содержимое репозитория
* `Acer Aspire 5742G.bin` — Оригинальный (чистый) дамп BIOS, считанный программатором (используйте для отката в случае проблем).
* `Acer Aspire 5742G-unlock.rom` — Модифицированный дамп с разблокированными меню `Advanced` (вместо `Security`) и `Power` (вместо `Information`), но названия вкладок остались прежними, просто с содержимым закрытых меню.

## 🧰 Необходимый инструментарий

Если вы хотите повторить процесс модификации самостоятельно, скачайте и соберите в отдельную папку следующие утилиты:
1. **AsProgrammer** (или NeoProgrammer / SiberiaProg) — программа для считывания и записи прошивки через программатор CH341A.
2. **UEFITool** — утилита для распаковки дампа BIOS, извлечения и обратной интеграции модулей.
3. **Universal IFR Extractor** — конвертер, превращающий логику меню BIOS (IFR) в читаемый текстовый формат.
4. **HxD** — Hex-редактор для низкоуровневой замены байтов логики и названий.
5. **H2OEZE (Insyde EZ H2O)** — сервисная утилита для замены графических элементов (заставки/Logo) в BIOS типа Insyde.

## ⚙️ Инструкция по самостоятельному моддингу BIOS

### Метод: Подмена масок (Form ID) в модуле SetupUtility

#### Шаг 1: Извлечение модуля логики
1. Откройте ваш считанный программатором дамп оригинального BIOS в программе **UEFITool**.
2. Нажмите `Ctrl+F`, перейдите во вкладку *Text* и введите в поиск строку `SetupUtility`.
3. В найденном модуле (обычно это раздел *PE32 image section*) кликните правой кнопкой мыши и выберите **Extract body**. 
4. Сохраните полученный файл (например, под именем `SetupUtility.bin`).

#### Шаг 2: Анализ IFR (Поиск масок меню)
1. Откройте утилиту **Universal IFR Extractor** и прогоните через неё извлеченный файл `SetupUtility.bin`. Утилита сгенерирует текстовый файл (`.txt`).
2. Откройте этот текстовый файл в любом редакторе и найдите секции `Form Set`. Выпишите шестнадцатеричные идентификаторы (Form ID) интересующих нас вкладок.
   * *Для данной модели (и многих аналогичных плат Acer) стандартные ID выглядят примерно так:*
     * Вкладка `Information` = `Form ID: 0x01`
     * Вкладка `Main` = `Form ID: 0x02`
     * Вкладка `Security` = `Form ID: 0x03`
     * Скрытая инженерная вкладка `Advanced` = `Form ID: 0x06`
     * Скрытая вкладка расширенного питания `Power` = `Form ID: 0x07`

#### Шаг 3: Подмена логики (Редактирование в HxD)
Наша цель — заставить BIOS загружать логику `Advanced` при клике на стандартную вкладку `Security`, и логику `Power` при клике на вкладку `Information`.
1. Откройте файл `SetupUtility.bin` в Hex-редакторе **HxD**.
2. Найдите в коде последовательность байтов, отвечающую за вызов вкладок главного меню (цепочка вызовов Form ID из предыдущего шага).
3. **Подмена Security:** Замените байт вызова Security (`03`) на байт маски Advanced (`06`).
4. **Подмена Information:** Замените байт вызова Information (`01`) на байт маски Power (`07`).

> 🛑 **КРИТИЧЕСКИ ВАЖНО:** Общий размер файла `SetupUtility.bin` не должен измениться ни на один байт! Используйте только функцию замены с перезаписью (Overwrite). Ни в коем случае не используйте вставку со сдвигом (Insert), иначе вы сломаете структуру модуля.

#### Шаг 4: Косметическое переименование вкладок (Опционально)
Этот шаг нужен, чтобы в самом BIOS вместо названий старых вкладок отображались новые. Текст хранится в формате Unicode.
1. В HxD нажмите `Ctrl+F`, перейдите на вкладку **Hex-values** (Шестнадцатеричные значения).
2. **Меняем Security на Advanced:**
   * Ищем HEX-строку: `53 00 65 00 63 00 75 00 72 00 69 00 74 00 79 00` (Слово S.e.c.u.r.i.t.y, 16 байт).
   * Выделяем эту строку целиком и делаем «Вставить с заменой» (Paste Write) на код: `41 00 64 00 76 00 61 00 6E 00 63 00 65 00 64 00` (Слово A.d.v.a.n.c.e.d, 16 байт).
3. **Меняем Information на Power:**
   * Ищем HEX-строку: `49 00 6E 00 66 00 6F 00 72 00 6D 00 61 00 74 00 69 00 6F 00 6E 00` (Слово I.n.f.o.r.m.a.t.i.o.n, 22 байта).
   * Делаем «Вставить с заменой» на код: `50 00 6F 00 77 00 65 00 72 00 20 00 20 00 20 00 20 00 20 00 20 00` (Слово P.o.w.e.r + 6 пробелов).
   * *Примечание: Пробелы (`20 00`) в конце строго обязательны! Они подгоняют короткое слово Power под длину изначального слова Information (22 байта). Если этого не сделать, таблица строк сместится и сломается.*

*(Внимание: в некоторых версиях BIOS текстовые названия дублируются во внутренних системных таблицах. Если после прошивки визуальные названия остались старыми, сама логика инженерных меню всё равно будет работать корректно).*

#### Шаг 5: Предварительная сборка модифицированного BIOS (UEFITool)
Перед заменой логотипа необходимо собрать измененный модуль логики обратно в файл прошивки.
1. Сохраните все изменения в пропатченном файле `SetupUtility.bin` в HxD.
2. Возвращайтесь в **UEFITool** (где у вас открыт оригинальный дамп BIOS).
3. Снова найдите секцию `SetupUtility`, кликните правой кнопкой мыши -> **Replace body** и выберите ваш модифицированный файл `.bin`.
4. Сохраните итоговый образ BIOS: `File` -> `Save image file` (назовите его, например, `mod_bios.rom`).

#### Шаг 6: Замена загрузочного логотипа через H2OEZE (Опционально)
Персонализация BIOS кастомной картинкой при включении. Выполняется **строго с файлом `mod_bios.rom`**, полученным на предыдущем шаге!
1. Запустите утилиту **H2OEZE**, в верхнем меню выберите `File` -> `Load ROM` и откройте ваш `mod_bios.rom`.
2. В боковом/верхнем меню перейдите в раздел `Components`, затем выберите `Logo`.
3. В открывшейся вкладке нажмите `Browse` и выберите подготовленное изображение.
   * **Требования к изображению:** Разрешение строго `1024x768`, формат **ТОЛЬКО BMP**. 
   * **Глубина цвета:** Изображение должно быть сохранено как `16-цветный рисунок (*.bmp;*.dib)`. Сделать это можно в стандартном редакторе Paint при сохранении файла (Сохранить как -> Тип файла).
   * **Вес файла:** Итоговый файл `.bmp` должен весить **строго менее 900 КБ**, иначе модуль BIOS не сможет его интегрировать из-за нехватки свободного места.
4. Пролистайте окно утилиты немного вниз и нажмите кнопку `Apply`. Появится диалоговое окно с предложением конвертировать формат/сжать изображение под стандарты Insyde — **обязательно нажмите `Yes`**. (Если нажать No, при загрузке логотип может не отображаться).
5. Нажмите `File` -> `Save`, чтобы окончательно сохранить прошивку с новой логикой и новым логотипом.

#### Шаг 7: Финальная прошивка
1. Откройте **AsProgrammer**.
2. Очистите чип памяти на материнской плате.
3. Откройте ваш финальный `mod_bios.rom` и запишите его. 
4. После записи обязательно дождитесь окончания верификации. Готово! Можно собирать и запускать ноутбук.
